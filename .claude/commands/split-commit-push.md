現在のすべての変更（staged, unstaged, untracked）を確認し、論理的に適切な単位に分割してコミットし、リモートに push してください。

## 手順

1. `git status` と `git diff --stat` で変更内容の全体像を把握する
2. `git log --oneline -5` で直近のコミットメッセージのスタイルを確認する
3. 変更を論理的なまとまりごとにグループ分けする（例: リファクタリング、新機能、バグ修正、ドキュメント、設定変更など）
4. 各グループごとに適切なファイルを `git add` してコミットする
5. コミットメッセージは英語の要約1行 + 空行 + 日本語の説明で書く
6. すべてのコミットが終わったら `git status` でワーキングツリーがクリーンなことを確認する
7. `git push` でリモートに反映する

## コミットメッセージの形式

```
英語の要約（1行、命令形）

日本語での変更内容の説明。
何を変えたか、なぜ変えたかを簡潔に書く。

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
```

## 分割の指針

- 関連するファイル変更は1つのコミットにまとめる
- 異なる目的の変更は別コミットに分ける
- ファイルのリネーム/移動とその中身の修正が連動する場合は同一コミットにする
- ドキュメントの追加・修正は独立したコミットにする
- 1コミットで済む場合は無理に分割しない
